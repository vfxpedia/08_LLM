# 02. 제안 배경 및 필요성

## 2.1 Why ZIPFIT?

### 2.1.1 공공주거 공고는 복잡하고 구조가 달라서, 사람만 보고 이해하기엔 너무 어렵기 때문이다

**문제 상황**:

공공주택(LH, SH, GH) 공고는 PDF 형태로 제공되며, 각 기관마다 구조와 양식이 다릅니다. 일반 국민이 이를 직접 읽고 이해하려면 다음과 같은 어려움이 있습니다:

1. **문서 길이**: 공고문 한 건당 평균 20~50페이지
2. **전문 용어**: "선계약후검증", "묵시적갱신", "계약보증금", "입주자격완화" 등
3. **복잡한 조건**: 소득 기준, 거주 요건, 우선순위 등이 복잡하게 얽혀있음
4. **산재된 정보**: 자격 요건, 신청 방법, 필요 서류, 일정 등이 문서 곳곳에 흩어져 있음

**실제 사례**:

```
"선계약후검증, 동호지정" 방식의 공고란?
→ 일반인은 이 용어만 봐서는 무슨 뜻인지 알 수 없음
→ 공고문 전체를 읽어야 이해 가능
→ 시간과 노력이 많이 소요됨
```

**ZIPFIT의 해결책**:

LLM이 공고문 전체를 읽고 분석하여, 사용자에게 필요한 정보만 쉽게 설명합니다.

```
사용자: "선계약후검증이 뭐야?"
ZIPFIT: "계약을 먼저 하고, 나중에 자격을 확인하는 방식입니다.
       빠르게 입주할 수 있지만, 자격이 안 맞으면 계약이 취소될 수 있어요."
```

### 2.1.2 LLM만이 비정형 문서를 읽고, 맥락을 이해하며, 사람처럼 질문에 답할 수 있기 때문이다

**기존 검색 방식의 한계**:

1. **키워드 검색**: "행복주택"을 검색하면 모든 행복주택 공고가 나열될 뿐, 내가 신청할 수 있는지는 알 수 없음
2. **필터링**: 지역, 유형 등으로 필터링 가능하지만, 복잡한 자격 요건은 자동으로 판단 불가
3. **FAQ**: 일반적인 질문에만 답변 가능, 개인 맞춤형 답변 불가

**LLM의 강점**:

1. **비정형 문서 이해**: PDF, 표, 이미지 등 다양한 형식의 문서를 읽고 이해
2. **맥락 파악**: 사용자의 이전 질문을 기억하고, 맥락에 맞는 답변 제공
3. **자연스러운 대화**: 딱딱한 명령어가 아닌, 일상 대화처럼 소통
4. **개인 맞춤형 답변**: 사용자의 조건(나이, 소득 등)을 고려하여 답변

**ZIPFIT의 접근**:

LLM + RAG(Retrieval-Augmented Generation) 결합으로:
- LLM의 자연어 이해 능력 활용
- RAG로 실제 공고문 참조하여 정확성 보장
- 할루시네이션(환각) 방지

```
사용자: "28살인데 서울에서 혼자 사는 청년이야. 행복주택 신청할 수 있어?"
ZIPFIT: (사용자 정보 분석 + 공고문 검색 + 자격 요건 확인)
       "네, 신청 가능합니다. 서울 지역 행복주택 3건을 찾았어요.
       모두 만 19~39세, 무주택자 조건을 충족하시네요."
```

### 2.1.3 현재 서비스에는 어떤 문제가 있는가? 기존 플랫폼은 공고를 '나열'만 할 뿐, 내 조건에 맞는지 이해하거나 설명해주지 못한다

**기존 플랫폼의 한계**:

**1. LH 청약센터 (apply.lh.or.kr)**
- 장점: 공식 사이트, 정확한 정보
- 단점:
  - 공고를 단순 나열만 함
  - 복잡한 UI, 찾기 어려운 메뉴 구조
  - 자격 확인 기능 없음
  - 전문 용어 설명 부족

**2. 서울주택도시공사 (i-sh.co.kr)**
- 장점: 서울 지역 공고 집중
- 단점:
  - LH, GH와 별도 사이트 (통합 검색 불가)
  - 공고 비교 기능 없음
  - 대출 정보 연동 안 됨

**3. 경기주택도시공사 (gh.or.kr)**
- 장점: 경기 지역 공고 집중
- 단점:
  - LH, SH와 별도 사이트 (통합 검색 불가)
  - 사용자 맞춤 추천 없음

**사용자 불편 사항**:

```
현재 사용자 여정:
1. LH 사이트 접속 → 공고 확인
2. SH 사이트 접속 → 공고 확인
3. GH 사이트 접속 → 공고 확인
4. 각 공고문 PDF 다운로드
5. 20~50페이지 공고문 읽기
6. 자격 요건 확인 (직접 계산)
7. 여러 공고 비교 (수작업)
8. 대출 정보 별도 검색
9. 신청 방법 별도 확인
→ 총 소요 시간: 2~3시간

ZIPFIT 사용자 여정:
1. ZIPFIT 접속 → AI 대화
2. 자격 자동 확인
3. 맞춤 공고 추천
4. 공고 비교 및 설명
5. 대출 정보 제공
6. 신청 안내
→ 총 소요 시간: 10~15분
```

## 2.2 제안 배경

### 2.2.1 주거 문제의 심각성

**통계 데이터** (예시):

1. **주거비 부담**:
   - 청년 1인 가구의 월 평균 주거비: 50만원 이상
   - 월 소득 대비 주거비 비율: 평균 30~40%

2. **주거 불안정**:
   - 청년층 주거 불안정 체감률: 70% 이상
   - 신혼부부 내 집 마련 평균 소요 기간: 10년 이상

3. **공공주택 수요**:
   - 연간 공공주택 공급: 약 10만 호
   - 평균 경쟁률: 5:1 ~ 20:1 (인기 지역)

**문제점**:

정부가 공공주택을 공급하고 있지만, 정보 접근성 부족으로 많은 국민들이 혜택을 받지 못하고 있습니다.

### 2.2.2 정보 격차 (Information Gap)

**디지털 격차**:

1. **고령층**: 복잡한 웹사이트 사용 어려움
2. **저소득층**: 정보 탐색에 필요한 시간과 자원 부족
3. **다문화 가정**: 한국어 이해도 부족

**정보 비대칭**:

- 정보를 잘 아는 사람: 여러 공고를 비교하고 최적의 선택
- 정보가 부족한 사람: 기회를 놓치거나 부적격 신청

**ZIPFIT의 역할**:

모든 국민이 동일한 수준의 정보에 접근하고, AI의 도움을 받아 최적의 선택을 할 수 있도록 지원합니다.

### 2.2.3 공공 데이터 활용의 필요성

**공공 데이터 현황**:

정부는 data.go.kr을 통해 공공주택 관련 데이터를 API로 제공하고 있습니다:
- 한국토지주택공사 분양임대공고문 조회 서비스
- 경기주택도시공사 GH주택청약 모집정보
- 서울주택도시공사 청약정보

**문제점**:

API는 제공되지만, 일반 국민이 직접 활용하기는 어렵습니다. 기술적 지식이 필요하고, 여러 API를 통합해서 사용해야 하기 때문입니다.

**ZIPFIT의 접근**:

공공 API를 활용하여 데이터를 수집하고, LLM으로 가공하여 사용자 친화적인 형태로 제공합니다.

## 2.3 필요성

### 2.3.1 기술적 필요성: LLM + RAG의 결합

**왜 LLM인가?**

1. **자연어 이해**: 사용자의 질문을 이해하고 맥락에 맞는 답변
2. **문서 분석**: 비정형 PDF 문서를 읽고 핵심 정보 추출
3. **쉬운 언어로 설명**: 전문 용어를 일반인이 이해할 수 있는 언어로 변환

**왜 RAG인가?**

1. **정확성**: 실제 공고문을 참조하여 사실에 기반한 답변
2. **할루시네이션 방지**: LLM이 잘못된 정보를 생성하는 것을 방지
3. **출처 명시**: 답변의 근거가 되는 문서와 페이지 번호 제공

**LLM + RAG의 시너지**:

```
LLM만 사용:
사용자: "행복주택 보증금이 얼마야?"
LLM: "보통 1000~2000만원 정도입니다." (부정확할 수 있음)

LLM + RAG:
사용자: "마포구 행복주택 보증금이 얼마야?"
RAG: 공고문에서 "마포구 공덕동 행복주택" 검색
LLM: "마포구 공덕동 행복주택 보증금은 1,200만원입니다.
     (출처: LH_마포공덕_행복주택.pdf p.5)"
```

### 2.3.2 사회적 필요성: 주거 복지의 실현

**주거권의 보장**:

헌법 제35조: "모든 국민은 건강하고 쾌적한 환경에서 생활할 권리를 가지며, 국가는 이를 위하여 노력하여야 한다."

공공주택은 주거권 실현의 핵심 수단이지만, 정보 접근성 문제로 실효성이 떨어지고 있습니다.

**ZIPFIT의 사회적 가치**:

1. **주거 불평등 완화**: 모든 국민이 공평하게 정보에 접근
2. **사회 안전망 강화**: 취약계층의 주거 안정 지원
3. **정부 정책 효과 증대**: 공공주택 공급 효율성 향상

### 2.3.3 경제적 필요성: 시간과 비용 절감

**개인 차원**:

- 공고 검색 시간: 2~3시간 → 10~15분 (90% 절감)
- 부적격 신청 방지: 시간과 비용 낭비 방지
- 최적의 공고 선택: 주거 비용 절감

**사회 차원**:

- 공공기관 업무 부담 경감: 부적격 신청 감소
- 행정 효율성 향상: 자격 확인 자동화
- 공공주택 입주율 향상: 적격자 매칭 최적화

### 2.3.4 기술적 필요성: 원스톱 서비스

**분산된 서비스의 문제**:

현재 사용자는 여러 사이트를 방문해야 합니다:
1. LH 사이트: 공고 확인
2. SH 사이트: 공고 확인
3. GH 사이트: 공고 확인
4. 주택도시기금 사이트: 대출 정보
5. 각 은행 사이트: 대출 상담
6. 부동산 커뮤니티: 후기 확인

**ZIPFIT의 원스톱 서비스**:

하나의 플랫폼에서 모든 과정 완결:
1. 공고 검색
2. 자격 확인
3. 공고 비교
4. 신청 안내
5. 대출 정보
6. 계약 지원 (향후)
7. 커뮤니티 리뷰 (향후)

## 2.4 기대 효과

### 2.4.1 사용자 측면

**시간 절약**:
- 공고 검색 및 분석: 2~3시간 → 10~15분

**정확성 향상**:
- 자격 요건 자동 확인으로 부적격 신청 방지
- 실제 공고문 기반 정확한 정보 제공

**편의성 증대**:
- 복잡한 웹사이트 탐색 불필요
- 자연스러운 대화로 정보 획득

**의사결정 지원**:
- 여러 공고 비교 분석
- 장단점 명확히 제시

### 2.4.2 사회 측면

**주거 복지 체감도 향상**:
- 공공주택이 "어렵고 복잡한 절차"에서 "쉽고 편리한 서비스"로 전환
- 공공주택 신청률 증가 → 주거 안정 기여

**디지털 격차 해소**:
- 고령층, 저소득층, 다문화 가정의 정보 접근성 향상
- 대화형 인터페이스로 기술 장벽 낮춤

**정보 비대칭 완화**:
- 모든 국민이 동일한 수준의 정보 접근
- 공평한 기회 제공

**사회 안전망 강화**:
- 주거 취약계층의 안정적 주거 지원
- 청년, 신혼부부, 저소득층 등 다양한 계층 지원

### 2.4.3 공공기관 측면

**업무 효율성 향상**:
- 부적격 신청 감소로 심사 업무 부담 경감
- 자격 확인 자동화로 행정 비용 절감

**공공주택 입주율 향상**:
- 적격자 매칭 최적화
- 빈집 발생 감소

**정책 효과 증대**:
- 공공주택 공급 정책의 실효성 향상
- 국민 만족도 증가

**데이터 활용 가치 증대**:
- 공공 데이터(data.go.kr) 활용 사례 창출
- 공공 데이터 활용 촉진

## 2.5 시장 분석

### 2.5.1 잠재 사용자 규모

**주요 타겟**:
1. 청년 (만 19~39세): 약 1,000만 명
2. 신혼부부 (혼인 7년 이내): 약 150만 가구
3. 저소득층 (기준 중위소득 100% 이하): 약 500만 가구

**총 잠재 시장**: 수백만 명

### 2.5.2 경쟁 서비스 분석

**직접 경쟁자**: 없음 (현재 LLM 기반 공공주택 정보 서비스 부재)

**간접 경쟁자**:
1. LH/SH/GH 공식 사이트: 공고 나열만, AI 기능 없음
2. 부동산 정보 플랫폼 (직방, 다방 등): 민간 매물 중심, 공공주택 정보 부족
3. 부동산 커뮤니티 (네이버 카페 등): 정보 신뢰성 낮음, 정리되지 않음

**ZIPFIT의 경쟁 우위**:
- LLM 기반 대화형 인터페이스
- RAG 기반 정확한 정보 제공
- 통합 플랫폼 (LH, SH, GH)
- 원스톱 서비스 (공고 ~ 대출)

### 2.5.3 비즈니스 모델 (향후 검토)

**무료 서비스** (기본):
- 공고 검색, 자격 확인, 기본 정보 제공

**프리미엄 서비스** (향후):
- 개인화 추천 알고리즘
- 우선 알림 서비스
- 전문가 상담 연결
- 커뮤니티 리뷰 프리미엄

**B2G (정부 대상)**:
- 공공기관과 협력
- 공식 플랫폼으로 통합
