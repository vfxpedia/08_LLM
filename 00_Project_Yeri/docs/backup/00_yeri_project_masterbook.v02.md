# 예리는 못 말려: 프로젝트 마스터북

## 1. 프로젝트 개요

이 프로젝트는 EEVE LLM을 중심 기술로 활용하며, 단순 대사 생성 수준을 넘어 예리의 감정 해석, 플레이어 답변 분석, 센스 평가, 그리고 맞춤형 반응 생성을 담당하는 **핵심 AI 로직**을 포함한다.

### 1.1 프로젝트 개념
‘예리는 못 말려’는 AI 기반 인터랙티브 연애 시뮬레이션 게임으로, 플레이어가 AI 캐릭터 ‘예리’와 시각적·감정적 상호작용을 통해 몰입형 경험을 얻는 것을 목표로 한다. 단순히 “차이점을 맞추는 게임”이 아니라, 감정과 센스를 평가하는 **하이브리드 감정형 인터랙션 시스템**이다.

### 1.2 핵심 경험
- **핵심 문장:** “오빠, 나 뭐 달라진 거 없어?”
- **핵심 감정:** 관찰력 + 감정 센스 + 몰입 대화의 결합.
- **목표:** 시각 정보와 언어 감정 피드백을 융합한 대화형 연애 콘텐츠.

### 1.3 게임 키워드
- AI 연애 시뮬레이션
- 시각 감정 퀴즈
- TTS/STT 음성 인터랙션
- SNS 밈 공유형 콘텐츠

## 2. 문서 구성 및 개발 순서 (Top → Bottom)

이 프로젝트는 **Runpod + Ollama EEVE** 기반으로 구축되며, LLM이 이미지 비교 결과, 사용자 답변, 감정 피드백을 통합 분석하여 예리의 반응과 점수를 생성한다.

### 2.2 문서 개발 순서

#### 1단계: 01_Game_Structure.md
- 오프닝, 이미지 표시 방식, TTS 타이밍, 플레이 흐름, 점수 구조 설계.
- **3턴(3·10·30초)** 타임라인 + 예리의 표정 변화 포함.

#### 2단계: 02_Score_System_Detail.md
- 감정 센스(60%), 관찰력(25%), 순발력(15%) 비율로 100점 만점 점수 시스템 확정.
- EEVE + VectorDB 감정 분석 및 점수 계산 로직 명세.

#### 3단계: 03_Character_Design.md
- 예리 캐릭터의 시각적/정서적 톤 확립.
- 시간 흐름에 따른 표정 변화(밝음 → 집중 → 울상 → 웃음) 로직 정의.

#### 4단계: 04_UI_UX_Design.md
- 상단 타임라인 애니메이션 설계.
- 카드 뒤집기 효과, 사운드 연출(샤라랄라 효과) 세부화.

#### 5단계: 05_Technical_Implementation.md
- EEVE LLM 작동 구조 및 Runpod 환경 세팅.
- TTS(gTTS/ElevenLabs), STT(Whisper) 연동.
- 이미지 비교 + VectorDB 감정 분석 처리 파이프라인 설계.

#### 6단계: 06_Production_Pipeline.md
- MVP → Prototype → Beta → Final 단계별 목표 및 일정 정의.
- 버전 관리 전략 및 QA 플로우 포함.

#### 7단계: 07_Marketing_SNS.md
- SNS 밈 카드 템플릿 제작.
- 공유 문구, 해시태그, 챌린지 이벤트 시나리오.

## 3. 전체 진행 원칙

> 각 단계의 산출물은 기능 검증, 사용자 피드백, QA 체크리스트를 통해 검증되며 피드백 루프는 최소 주 1회 수행된다.

1. **문서 모듈화:** 모든 문서는 독립적으로 관리하되, 상호 참조 가능하게 설계.
2. **Top → Bottom 순서:** 게임 구조 → 점수 로직 → 캐릭터 → UI/UX → 기술 → 마케팅 순으로 개발.
3. **피드백 루프:** 각 단계 완료 후 사용자(Jay)의 검토 및 컨펌 필수.
4. **확장성 보장:** 향후 ‘AI Love Coach Mode’ 및 ‘Voice Dating Mode’ 추가를 고려.
